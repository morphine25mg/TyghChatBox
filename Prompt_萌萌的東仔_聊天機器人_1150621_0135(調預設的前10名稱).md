## 二、 聊天室核心功能

1. **訊息推播與滾動**：最新訊息必須自動往上推，對話視窗需自動滾動聚焦至最底部。
2. **打字中動畫**：當機器人正在處理或呼叫 API 時，需顯示動態的「打字中...」動畫（如三個閃爍的圓點）。
3. **連結優化**：訊息內若包含 URL 網址，必須自動轉換為可直接點擊的超連結（`target="\_blank"`）。
4. **智慧輸入框優化**：在「請輸入您的問題...」輸入框中，同仁按下鍵盤 `Enter` 鍵等同點擊發送按鈕。
5. **動態快捷提問 (Chips)**：

   * 在對話框下方、輸入欄位上方，提供熱門問答 Top 10 的快捷鍵按鈕（樣式為 Chip 標籤）。
   * 熱門問題 Chips 區塊須**顯示可見的橫向滾動條**（不可隱藏），讓使用者明確知道可以左右滑動查看更多。
   * 熱門度排序邏輯：**以問答記錄（qa\_logs）中各問題關鍵字的出現頻率降冪排序，取前 10 名**；若問答記錄筆數**少於 5 筆**，則用種子資料補足至 10 個。**每次問答完成後，立即重新計算並更新 Chips 排序。**
   * **自動過濾與阻斷機制**：

     * 若某個常見問答的內容（答案）內沒有任何文字或連結（空值），則該項目不得出現在前 10 名。
     * 若該提問屬於「無法回答 / 系統未具備此知識」的紀錄，必須自動被過濾掉。
     * *冷啟動保護*：若問答記錄筆數少於 5 筆（含零筆），以種子資料（【在職進修】、【能力進階】、【抽血試管 】、【緊急應變】、【常見診斷】、【清消】、【開機密碼】、【轉病房帶幾罐】、【帳密 】、【垃圾分類】補足至 10 個顯示；一旦真實問答記錄超過 5 筆，立即改以真實頻率排序完全取代種子資料。
   * 點擊任一 Chip 需直接觸發該關鍵字發送。
6. **歡迎訊息規範**：

   * 歡迎訊息採**單行緊湊格式**：`👋 你好，我是萌萌的東仔！AICU 的知識小幫手`
   * 整張歡迎卡片高度**不超過 60px**，字體大小 13px，不換行，不顯示任何額外說明文字。

## 三、 知識庫檢索與三層式 AI 兜底機制

> ⚠️ \*\*核心架構說明\*\*：系統採「三層回答機制」，依序嘗試，任一層成功即停止往下。

```
【第一層】本地知識庫關鍵字比對
        ↓ 找不到
【第二層】Transformers.js 本地 AI 從知識庫全文抽取答案（免費、離線可用）
        ↓ 信心分數不足（< 0.3）或模型尚未載入
【第三層】Claude API 線上回答（需填入 API Key，選配）
        ↓ 未設定 API Key 或呼叫失敗
【阻斷】回覆「此系統未具備此知識，建議洽詢主管或相關單位喔」
         並寫入知識盲區日誌
```

### 1\. 第一層：本地知識庫關鍵字比對

* 比對使用者輸入與本地知識庫的標題、關鍵字、答案欄位。
* **文字搜尋必須忽略大小寫（Case-Insensitive）**。
* Token 命中率達 60% 以上即視為命中，最多回傳 3 筆最相關結果整合顯示。
* 若有多筆相符答案，應全數列出或整合給使用者。

### 2\. 第二層：Transformers.js 本地 AI 兜底（🆓 免費、不需 API Key）

#### 運作原理

* 使用 **`Xenova/distilbert-base-cased-distilled-squad`** 問答抽取模型。
* 模型透過 CDN 載入：

```html
  <script type="module">
    import { pipeline } from 'https://cdn.jsdelivr.net/npm/@xenova/transformers@2.17.2';
  </script>
  ```

* 將**知識庫所有條目的標題＋關鍵字＋答案**串接成一段大型 context 文字。
* 模型從這段 context 中「抽取」最可能回答問題的片段，而非憑空生成。

#### 初始化規範

* 頁面載入完成後，在**背景非同步預載**模型（不阻擋使用者操作）。
* 預載期間，在介面顯示小型提示：`🤖 AI 助理準備中...`（右上角或底部狀態列）。
* 預載完成後提示消失，改顯示：`✅ AI 助理就緒`，3 秒後自動隱藏。
* 若載入失敗（如網路問題），靜默跳過，直接降級至第三層或阻斷。

#### 呼叫規範

```javascript
// 【第二層】Transformers.js 從知識庫全文抽取答案
async function tryTransformersQA(userQuery) {
  if (!window.qaModel) return null; // 模型尚未就緒，跳過

  // 將所有知識庫條目串成 context（最多取前 3000 字，避免超出模型限制）
  const kb = getKB();
  const context = kb.map(item =>
    `${item.title}：${item.keywords || ''}。${item.answer}`
  ).join('\\n').substring(0, 3000);

  try {
    const result = await window.qaModel({
      question: userQuery,
      context: context
    });

    // 信心分數門檻：低於 0.3 視為沒有找到合適答案
    if (result.score < 0.3) return null;

    return `📖 ${result.answer}（AI 從知識庫摘錄）`;
  } catch (e) {
    return null; // 任何錯誤皆靜默跳過
  }
}
```

#### 答案標示規範

* Transformers.js 回答的訊息氣泡，需在答案末尾加上標籤：`（AI 從知識庫摘錄）`
* 此類回答**同樣顯示**👍👎⚠️ 三個滿意度按鈕。
* 回答者頭像與一般東仔回答相同（東元獅頭像）。

### 3\. 第三層：Claude API 線上備援（選配，需 API Key）

* 僅在第一層與第二層**皆無法回答**時才觸發。
* 請在程式碼中預留 API Key 欄位與 fetch 語法架構。
* Prompt 框架：限制 Claude 以 AICU 小幫手的語氣，用繁體中文簡短親切回答，100 字以內。
* Claude API 回答的訊息需加標籤：`🤖 AI 回答`

### 4\. 阻斷與知識盲區日誌

* 三層皆無法回答時，統一回覆：「此系統未具備此知識，建議洽詢主管或相關單位喔 🙏」
* 同時將該**搜尋關鍵字與精確時間**寫入後台「知識盲區紀錄」。

## 五、 後台管理系統（安全與分頁設計）

在介面右上方小圖示提供後台入口。

* **安全驗證**：進入後台前必須跳出密碼輸入框，輸入「`0717`」方可解鎖進入。
* **後台三大功能分頁**：

### 1\. 知識庫管理分頁

* **升級多格式檔案解析與擴充**：

  * 提供一個拖曳區（Drag and Drop）或檔案上傳鈕，支援 `.txt`、`.csv`、`.md`、`.pdf`、`.doc`、`.docx`、`.pptx`、`.xlsx`。
  * **`.xlsx` 解析規範**：使用 **SheetJS（xlsx.js）CDN 函式庫**解析 Excel 內容：

    * 讀取第一個工作表的所有資料列。
    * 欄位對應：**第一欄（A）= Question（標題）**、**第二欄（B）= Key words（關鍵字）**、**第三欄（C）= Answer（答案）**。
    * 每一列解析成一筆知識條目直接寫入知識庫，立即可被搜尋。
    * 上傳完成顯示「✅ 已成功匯入 N 筆知識條目」。
  * **其他二進位格式**（`.pdf`、`.doc`、`.docx`、`.pptx`）：擷取檔案名稱為標題，產生系統描述文字 `\[系統已載入 ${file.name}，大小 ${size} KB，以檔名作為搜尋關鍵字]` 作為可搜尋內容。
  * 同時支援直接手動輸入文字欄位。
  * 知識庫 支援同檔案上傳，回饋是否覆蓋，是，覆蓋，否，則取消上傳
* **Transformers.js 模型控制**（後台新增區塊）：

  * 顯示目前 AI 模型載入狀態：`未載入 / 載入中 / 就緒`。
  * 提供【🔄 重新載入 AI 模型】按鈕（當模型載入失敗時手動觸發）。
  * 顯示說明文字：「AI 兜底功能使用 Transformers.js，模型存於瀏覽器快取，首次使用需下載約 67MB，之後可離線運作。」
* **完整 CRUD 與即時同步**：

  * 下方條列顯示所有知識條目（顯示：標題、上傳更新日期、內容簡述）。
  * 每個條目具備【個別刪除】與【個別下載（匯出 .txt）】功能。
  * 新增或刪除後，前台搜尋與 Transformers.js 的 context 即時同步更新。

### 2\. 問答記錄與匿名統計分頁

* **問答記錄與基本看板**：

  * 完整保存每一則問與答（含時間戳記），管理員可手動刪除。
  * 即時統計：總問答數、有幫助次數、沒幫助次數、回報管理員次數、綜合滿意度（%）。
  * **新增統計項目**：第二層 AI 回答次數（Transformers.js 命中次數）、第三層 Claude API 回答次數。
* **擴充：追蹤回報與錯誤日誌區塊**：

  * **【回報管理員資料明細庫】**：條列回報記錄，提供一鍵「刪除/已修正」按鈕。
  * **【知識盲區紀錄】**：條列所有三層皆無法回答的搜尋關鍵字與時間。

### 3\. 系統設定分頁

* **護理長信箱變更**：可修改 `mailto:` 收件人信箱。
* **後台密碼修改**：可修改預設的「0717」登入密碼。
* **Claude API Key 設定**：

  * 提供輸入欄位填入 `sk-ant-...` 格式的 API Key。
  * 說明文字：「此為選配功能。API Key 僅存於瀏覽器 localStorage，不會傳送至任何伺服器。知識庫與 AI 兜底皆無法回答時才會呼叫。」
* **AI 兜底開關**：

  * 提供切換開關（Toggle）控制是否啟用 Transformers.js 第二層兜底。
  * 預設為**開啟**。
  * 說明：「關閉後，知識庫找不到答案將直接跳至 Claude API 或阻斷回覆。」

## 六、 技術與交付要求

1. 所有 HTML、CSS、JavaScript 撰寫在**單一 `.html` 檔案**中。
2. **必要 CDN 引用**（請置於 `<head>` 內）：

&#x20;   ```html
    <!-- SheetJS：xlsx 檔案解析 -->
    <script src="https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js"></script>

    <!-- Transformers.js：本地 AI 模型（以 ES Module 方式引用，置於 JS 邏輯內） -->
    <!-- import { pipeline } from 'https://cdn.jsdelivr.net/npm/@xenova/transformers@2.17.2'; -->
    ```

3. **Transformers.js 載入規範**：

   * 必須以 `<script type="module">` 方式引用，避免阻塞頁面渲染。
   * 模型預載入後存於 `window.qaModel`，供全域呼叫。
   * 第一次下載約 67MB，存於瀏覽器 Cache Storage，之後重新整理不需重新下載。
4. 前端資料以 `localStorage` 持久化保存（知識庫、問答紀錄、統計、回報明細、盲區紀錄、設定值）。
5. **程式碼註解重點**：

   * 三層回答機制的流程控制邏輯
   * Transformers.js pipeline 初始化與 context 串接方式
   * 信心分數（score）門檻判斷
   * SheetJS xlsx 欄位解析對應
   * 動態 Top 10 Chips 篩選演算法
   * mailto 字串串接
